# 211_rs — RSスキャナー（相対力戦略）

日本株ユニバース（230銘柄）を対象に、ベンチマーク（日経225・TOPIX）に対する相対力（RS）を計算し、強い銘柄をDiscordに通知するスキャナーです。

---

## ファイル構成

| ファイル | 役割 |
|---|---|
| `rs_scanner.py` | RSスキャン実行・Discord通知 |
| `rs_monitor.py` | 保有ポジション監視・エグジット通知 |
| `universe230.csv` | スキャン対象銘柄リスト（230銘柄） |
| `requirements.txt` | 依存ライブラリ |

| `.github/workflows/rs_monitor.yml` | モニタリング用GitHub Actionsワークフロー |

---

## シグナル定義

### [A] RS水準
いずれかのRS期間（5日・10日・20日）で RS > 1.15  
→ベンチマークを15%以上アウトパフォームしている状態

### [B] RS転換（底打ち反転）
直近4日のRSウィンドウで底打ちから反転している状態  
条件：直近RS > 1本前RS かつ 底値 < ウィンドウ起点

### [C] 隠れ強気
株価5日リターン < 0（株価は下落）かつ RS20日 > 1.10  
→株価が下がっているのにベンチマーク比では強い状態

### 通知条件
**[A]+[B]+[C] の三拍子揃った銘柄のみ通知**（バックテスト最適化済み）

## バックテスト結果

### 検証条件

| 項目 | 設定値 |
|---|---|
| 検証期間 | 10年（2015〜2025年） |
| ユニバース | 230銘柄 |
| エントリー | シグナル翌日の終値 |
| ストップ判定 | **日足安値ベース**（終値ではなく日中の安値で判定） |
| 除外月 | 3月（唯一の平均マイナス月） |
| 決算シーズン除外 | 5・8・11・2月の1〜20日 |
| 配当権利落ち除外 | 9月末（28〜30日をまたぐ保有） |

### パラメータ最適化結果（保有日数 × 損切り水準）

| # | 保有日数 | 損切り | 件数 | 勝率 | PF | 平均損益 |
|---|---|---|---|---|---|---|
| ★1 | 20日 | なし | 282,723件 | 55.7% | 1.52 | +1.36% |
| ★2 | 15日 | なし | 294,431件 | 55.2% | 1.45 | +1.05% |
| ★3 | 20日 | -10% | 282,723件 | 54.0% | 1.44 | +1.20% |

> **重要な知見：タイトな損切りは逆効果**  
> RS強銘柄は一時的に押しても戻りやすく、-1〜-3%の損切りはPFを下げる。  
> 損切りなし or -8〜-10%（暴落保険のみ）が最適。

### シグナル種別成績（保有20日 / 損切りなし）

| シグナル | 件数 | 勝率 | PF | 平均損益 |
|---|---|---|---|---|
| [A] のみ | 35,487件 | 54.1% | 1.39 | +1.10% |
| [A]+[B] | 164,134件 | 55.1% | 1.48 | +1.24% |
| [A]+[C] | 14,361件 | 57.1% | 1.52 | +1.48% |
| **[A]+[B]+[C]** | **68,741件** | **57.9%** | **1.69** | **+1.77%** |

→ **[A]+[B]+[C] に絞ることでPF・勝率ともに最良**

### 月別フィルターの効果

| 除外設定 | 全体PF | [A]+[B]+[C] PF |
|---|---|---|
| 3月のみ除外（旧設定） | 2.22 | 3.06 |
| 3月除外 + 決算・配当除外（現設定） | 1.52 | 1.69 |

> ※旧設定は終値ベースのストップ判定による過大評価。現設定が現実的な値。

## セクター出遅れ検出（補助機能）

BNF氏の手法に基づく「セクター全体が上昇しているが出遅れている個別銘柄」を検出します。

**検出条件：**
- セクター平均5日リターン > +2%
- 個別銘柄リターン < セクター平均の50%（出遅れ）
- 出来高が直近20日平均の1.1倍以上（資金流入確認）

> ※この機能はバックテスト未実施。RS主軸シグナルの補助参考情報として使用。

## Discord通知フォーマット

### スキャン結果通知

```
🔥 **銘柄名**（ティッカー）  [最優先]  ⏱目標20日保有
   現在値  5日リターン  RS20=X.XX
   📌 参考: エントリー範囲 | 🛑 損切目安: XXX円（-10%）
📎 ticker|rs|entry_price|stop_loss|name
```

`📎` 行は `301_portfolio_summary/discord_poll.py` と連動。  
🛒 リアクションで `positions.json` に自動登録されます。

### モニタリング通知（rs_monitor.py）

- 保有20営業日到達 → 📤 エグジット通知
- 現在値が損切りラインを下回った → 📤 損切り通知
- 毎日の保有状況サマリーを送信

---

## 301_portfolio_summary との連動フロー

```
1. rs_scanner.py 実行
   → Discord に 🛒メッセージ（📎形式）送信

2. ユーザーが 🛒 リアクション
   → discord_poll.py が positions.json に追加
      （リポジトリ: trading-for-nouka/211_rs）

3. rs_monitor.py 実行（毎平日16:00）
   → エグジット条件を確認
   → 通知 + positions.json から削除

4. portfolio_summary.py（301リポジトリ）
   → 残ポジションをサマリー表示
```
